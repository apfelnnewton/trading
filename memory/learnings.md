## 교훈 1: 추정치 recPrice는 절대 금지
**사건:** 2026-04-24 현대제철 recPrice를 47,000원(4/9 종가 39,450원 × +19.85% 추정)으로 설정. 실제 시작가 41,900원과 12% 괴리. 사용자 시작가 제공 없었으면 miss로 오집계될 뻔함.

**교훈:** 
- 아무리 합리적으로 보이는 추정 공식도, **당일 실거래 가격과 비교하면 틀린 게 정상**
- "추정", "~적용 시", "역산" 같은 단어가 supply/reason에 들어가면 그 recPrice는 **무조건 폐기**
- 07:35 시점에 전일 종가를 확실히 모르면 `null`로 두고 사용자가 알려줄 때까지 기다리는 게 정답

**규칙화:** 규칙 9

## 교훈 2: 실적 발표는 예측 불가능한 '셀 온 뉴스' 지뢰밭
**사건:** 2026-04-23 SK하이닉스 역대급 1Q 실적(매출·영업익 사상 최대) 발표에도 불구하고 주가 하락. KB금융·HD한국조선해양·POSCO홀딩스 모두 실적 D-Day였고, 적중률 50% / 평균 -1.75%로 저조. 그중 KB금융은 손절선 -5.4% 정확히 터치.

**교훈:**
- 컨센서스 상회해도 "이미 반영됐다"며 파는 세력 항상 존재
- **4종목 모두 실적 D-Day는 노출 과다.** 섹터 로테이션·수급 쌍끌이·차트 돌파·정책 이슈 같은 다른 촉매와 섞어야 함
- 실적 발표 당일 ±5% 손절선은 너무 타이트 → 정상 변동성(±5~7%)에 걸려 무용한 손절

**규칙화:** 규칙 1 (4종 중 D-Day 최대 2개), 규칙 3 (D-Day 손절폭 ±7~10% + 비중 절반)

## 교훈 3: 종목명이 비슷하면 무조건 코드부터 확정
**사건:** 2026-04-23 HD한국조선해양(009540)과 HD현대중공업(329180)을 혼동해 recPrice 오기재. POSCO홀딩스(005490)와 포스코퓨처엠(003670)도 유사 사례 다수.

**교훈:**

## 교훈 4: 코스피 사상 최고 돌파 직후는 별점 인플레이션 주의
**사건:** 2026-04 코스피 6,400 돌파 직후 4종목 모두 ⭐5로 설정했다가 과도한 기대 유발.

## 교훈 5: 사용자 실제 데이터(종가·시작가)가 최고 권위
**사건:** 2026-04-24 KT&G를 14:34 WebSearch 기반으로 158,900원(-1.07% 손절권)으로 판단 → 실제 15:30 종가 176,200원(+3.65% hit). 만약 실제로 14:34에 팔았으면 기회 손실.

**교훈:**
- WebSearch는 **지연·일부 반영·구식 데이터** 섞임. 특히 마감 임박 시간대에 불안정
- 사용자가 직접 보는 네이버금융·다음·MTS 실시간 종가·시가가 **언제나 1순위**
- 사용자 제공 값이 들어오면 기존 WebSearch 기반 판단을 **즉시 덮어쓰기**

**규칙화:** 규칙 6, 규칙 9

## 교훈 6: HTML 파일 닫힘 태그는 신성불가침
**사건:** 2026-04-24 Edit 도구 + 파일시스템 sync 불일치로 bash 측에서 `</script></body></html>` 없는 truncated 파일 관찰. 이대로 푸시하면 GitHub Pages 완전 깨짐.

**교훈:**
- 편집 후 **항상** `grep -c "</script>\|</body>\|</html>"` 로 3 확인
- **항상** `node --check`로 스크립트 블록 문법 검증
- 깨짐 발견 시 GitHub 온전한 이전 커밋 clone 후 Python으로 edit 재적용 → 푸시

**규칙화:** 규칙 7
